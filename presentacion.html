Gestor de Guías Envía

Aplicación local en Python para automatizar el proceso diario de guías de envíos.

Esta herramienta permite importar planillas descargadas manualmente, consolidar las guías, conservar la información hasta que el usuario decida borrarla, generar el archivo de trabajo y permitir editar OPERADOR, ESTADO y CAUSAL de cada guía.

📥

Importar

Importa planillas .xls / .xlsx descargadas manualmente, sin perder la información ya guardada.

🗄️

Consolidación acumulativa

Actualiza guías repetidas por número de GUIA sin borrar datos anteriores ni ediciones previas.

✏️

Edición visual

Interfaz gráfica (Tkinter) para editar operador, estado y causal — individual o masivamente.

📈

Informes

Genera informes generales y por operador a partir del consolidado diario.

Cómo iniciar

Da doble clic en INICIAR_GESTOR.bat. El primer inicio crea el entorno, instala el programa y genera la configuración local automáticamente.

Menú principal

Opciones disponibles al ejecutar INICIAR_GESTOR.bat

1

Importar planilla descargada

Selecciona el archivo .xls o .xlsx descargado y, opcionalmente, una fecha. El sistema procesa la planilla y actualiza la base local.

2

Abrir editor de operador, estado y causal

Abre la interfaz gráfica para revisar y editar guías individualmente o por lotes.

3

Exportar consolidado

Genera nuevamente el Excel final con toda la información guardada hasta el momento.

4

Generar informes

A partir de un consolidado .xlsx, genera informes generales y por operador.

5

Borrar toda la información guardada

Elimina por completo la base local. Requiere escribir BORRAR para confirmar — acción irreversible.

6

Salir

Cierra el programa.

Guías activas (estado N)

Solo quedan en el consolidado diario las guías cuyo estado de movimiento es N. El resto se separa automáticamente.

Copia de movimientos

Las filas con estado distinto de N se guardan en un archivo aparte: movimientos otros estado DD mes.xlsx.

Normalización de GUIA

El número de guía se guarda sin guiones y sin cero inicial, para mantener un formato consistente.

Conservación de ediciones

OPERADOR, ESTADO y CAUSAL editados manualmente se conservan al volver a importar.

Estructura del consolidado diario

Columnas que conserva el Excel final

PLANILLA SERVICIO GUIA UNID TIPO DE SERVICIO DESTINATARIO MUNICIPIO VALOR OPERADOR ESTADO CAUSAL FECHA INGRESO

Las columnas en azul (OPERADOR, ESTADO, CAUSAL) son las que se editan desde la interfaz gráfica.

Reglas aplicadas desde los reportes descargados

Columna / ReglaComportamiento
SERVICIOToma la marca TG (por ejemplo RR) cuando existe.
TIPO DE SERVICIOToma el tipo PT, MT, DE, etc.
GUIASe guarda sin guiones y sin cero inicial.
VALORSi el valor es cero, se muestra como $ -.
OPERADOR / ESTADO / CAUSAL / INGRESOQuedan vacíos al consolidar (se llenan luego desde el editor).
ESTADO / ESTADO MOVIMIENTOSolo se consolidan las filas con valor N.
Otros estadosSe guardan aparte en movimientos otros estado DD mes.xlsx.

Instalación y ejecución

Pasos para preparar el entorno en Windows

1

Abrir la carpeta del proyecto

Ubica la carpeta gestor_guias_envia en el explorador de archivos.

2

Ejecutar INICIAR_GESTOR.bat

Doble clic. La primera vez crea el entorno virtual .venv, instala las dependencias y genera config/settings.toml a partir del archivo de ejemplo.

3

Elegir una opción del menú

Importar, editar, exportar, generar informes o borrar datos según se necesite.

4

Editor visual

ABRIR_EDITOR.bat abre directamente la interfaz gráfica de edición.

Seguridad y conservación de datos

Cómo se protege la información local

🗄️

Base de datos local

Toda la información queda en data/database/guias.db, en el equipo del usuario.

♻️

No se pierde al importar

Importar nuevas planillas no borra la información ya guardada ni las ediciones previas.

⚠️

Borrado solo manual

Solo se elimina con borrar-datos --confirmar, escribiendo BORRAR para confirmar.

🙈

Archivos excluidos del control de versiones

Configuración local, base de datos y archivos de salida no se incluyen en git.
